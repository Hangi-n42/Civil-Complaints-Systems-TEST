# 0002 PR Review Tagging Convention

- Status: Accepted
- Date: 2026-04-30
- Decision makers: Team, repository maintainer
- Related issue / PR: Collaboration setup PR

## Context

PR feedback needs a simple severity signal so contributors can triage comments quickly.

## Decision

- Use `[MUST]` for blocking feedback.
- Use `[SHOULD]` for recommended but non-blocking feedback.
- Keep each comment focused and actionable.

## Consequences

- Review triage becomes faster.
- Review quality becomes more consistent across contributors.
